<h1>Overview</h1>
<p style="margin-left: 30.0px;">The modeling process is designed to occur in nine phases, with responsibility migrating among the Data Analysts, the Researchers, and the Software Development team. Ideally, one of each (a Researcher, an Analyst, and a Software Developer) will be assigned to each model.</p>

<p style="margin-left: 120.0px;"><strong>Generic Modeling Process</strong></p>
<h1>Intervention Report&nbsp;<ac:structured-macro ac:name="anchor" ac:schema-version="1" ac:macro-id="f6b1b220-9398-4a67-bc1c-bd410fd358ff"><ac:parameter ac:name="">InterventionReport</ac:parameter></ac:structured-macro></h1>
<p style="margin-left: 30.0px;">The&nbsp;<em>Intervention Report</em>&nbsp;is intended as a living record of model status. It contains the major decisions and rationale as they exist at the end of each phase.</p>

<h1>Model Development Workflow</h1>
<p style="margin-left: 30.0px;">The team has developed an eight phase model development workflow with phase-specific outputs.</p>
<p style="margin-left: 30.0px;"><ac:image ac:height="150"><ri:attachment ri:filename="Model_Workflow1.png" /></ac:image></p>
<h2 style="margin-left: 30.0px;">Roles</h2>
<h3 style="margin-left: 60.0px;">Researcher</h3>
<p style="margin-left: 60.0px;">The Researcher leads the model development process by guiding the background research, conceptualizing possible modeling strategies, validating those strategies with domain experts, guiding the conceptual development of software that will run the model, and generating analytics for model inputs and outputs. The researcher may develop tools in support of these processes.</p>
<h3 style="margin-left: 60.0px;">Engineer</h3>
<p style="margin-left: 60.0px;">The Engineer produces all of the code to run the model, including the development of model components and observers, as well as the development of tools in support of model and input data analytics.</p>
<h2 style="margin-left: 30.0px;">Phase Descriptions</h2>
<h3 style="margin-left: 60.0px;">Phase 1: Background Research</h3>
<p style="margin-left: 60.0px;">Complete a <em>Systematic Search</em>&nbsp;and generate a&nbsp;<strong>ROUGH</strong>&nbsp;first draft of&nbsp;the&nbsp;<em><ac:link ac:anchor="InterventionReport"><ac:plain-text-link-body><![CDATA[Intervention Report]]></ac:plain-text-link-body></ac:link>.</em> See&nbsp;<ac:link><ri:page ri:space-key="COS" ri:content-title="Phase 1: Background Research for New Models" /></ac:link>&nbsp;</p>
<h3 style="margin-left: 60.0px;">Phase 2: Concept Model Development</h3>
<p style="margin-left: 60.0px;">Complete a <em>Systematic Review</em> and a <em>Concept Model Document</em>. See <ac:link><ri:page ri:space-key="COS" ri:content-title="Phase 2: Concept Model Development" /></ac:link>.</p>
<h3 style="margin-left: 60.0px;"><span>Phase 3: Data Extraction and Minimal Modeling</span></h3>
<p style="margin-left: 60.0px;">Generate a <em>Data Artifact</em> and update the <em>Intervention Report</em> to include the current status. See&nbsp;<ac:link><ri:page ri:space-key="COS" ri:content-title="Phase 3: Data Extraction and Data Modeling" /></ac:link>.</p>
<h3 style="margin-left: 60.0px;">Phase 4: Minimal Simulation Model Implementation</h3>
<p style="margin-left: 60.0px;"><em>This phase can progress simultaneously with Phase 3.</em>&nbsp;See&nbsp;<ac:link><ri:page ri:space-key="COS" ri:content-title="Phase 4: Minimal Simulation Model Implementation" /></ac:link>.</p>
<h3 style="margin-left: 60.0px;">Phase 5: Full Model Implementation</h3>
<p style="margin-left: 60.0px;">Add non-minimal components, validate results, and iteratively update until a final model is created. See&nbsp;<ac:link><ri:page ri:space-key="COS" ri:content-title="Phase 5: Full Simulation Model Implementation" /></ac:link>.</p>
<h3 style="margin-left: 60.0px;">Phase 6: Simulation Model Review</h3>
<p style="margin-left: 60.0px;">Results are presented to the full team for final validation. See&nbsp;<ac:link><ri:page ri:space-key="COS" ri:content-title="Phase 6: Model Review" /></ac:link>.</p>
<h3 style="margin-left: 60.0px;">Phase 7: Process Revision</h3>
<p style="margin-left: 60.0px;">The team discusses methods for iterative workflow improvement and brainstorms new tools facilitate future model development. See&nbsp;<ac:link><ri:page ri:space-key="COS" ri:content-title="Phase 7: Process Revision" /></ac:link></p>
<h3 style="margin-left: 60.0px;">Phase 8: Finalization</h3>
<p style="margin-left: 60.0px;">Models are archived and published. See&nbsp;<ac:link><ri:page ri:space-key="COS" ri:content-title="Phase 8: Model Finalization" /></ac:link>.</p>
<h3 style="margin-left: 60.0px;">Phase 9: Celebration</h3>
<p style="margin-left: 60.0px;">Pat self on back. Maybe get ice cream. See&nbsp;<ac:link><ri:page ri:space-key="COS" ri:content-title="Phase 9: Celebrate" /></ac:link>.</p>
